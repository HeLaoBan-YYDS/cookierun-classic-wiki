状态通过

sitemap 中 lastModified 永远是"今天"
影响中Google 可能逐渐降低对 lastModified 的信任度
证据lastModified: new Date()，每次请求都生成当前时间。

sitemap 与 HTML hreflang 中根 URL 尾斜杠不一致

证据 sitemap.ts 中 url = ${'$'}{siteUrl}${'$'}{locale === "en" ? "" : ...} 生成 https://cookierun-classic-wiki.wiki（无尾斜杠）； 而渲染出的 HTML hreflang x-default 为 https://cookierun-classic-wiki.wiki/（有尾斜杠）。
建议统一尾斜杠策略：在 next.config.mjs 显式设置 trailingSlash: false 或 true，并在 sitemap 与 canonical 中保持一致。

3. 国际化（hreflang / 多语言）审计

hreflang 标签结构完整
状态通过
证据渲染输出 5 个 hreflang：en、th、ko、ja、x-default，包含自引用（en 指向 /，th 指向 /th）和 x-default。
语言代码使用 ISO 639-1 标准格式
状态通过 en/th/ko/ja 均为合法语言代码
每个 locale URL 自引用 canonical
状态通过
说明/th/guide/... 的 canonical 指向 /th/guide/...，无跨 locale canonical 问题。
Accept-Language 自动检测会误导 Googlebot
影响严重页面行为不可预测，可能被错误重定向
证据 src/i18n/routing.ts 设置 localeDetection: true。审计过程中用 Chrome 访问 / 时被自动重定向到 /th，而访问 /en 时被重定向到 /。 Googlebot 不发送 Accept-Language 头，但其他爬虫与 CDN 缓存可能因此缓存错误的语言版本。
建议将 localeDetection 改为 false，仅依赖 hreflang 与显式语言切换器。
根路径 / 的重定向逻辑冲突
影响严重潜在的重定向循环 / 死代码
证据 src/app/page.tsx 写死 redirect("/en")，但 routing.ts 配置 localePrefix: "as-needed"（en 不带前缀）。 实际访问 /en 会被 next-intl 中间件重定向到 /，若此中间件失效则形成循环：/ → /en → /。
建议删除 src/app/page.tsx，让根路径 / 直接由 [locale] 路由处理（en 为默认 locale，无前缀）。
缺少 meta http-equiv="content-language"
影响中Bing 把 hreflang 视为弱信号，依赖 <html lang> 与 content-language
说明已有 <html lang="th">，建议补充 <meta http-equiv="content-language" content="th">（每个 locale 随之变化）。
sitemap 未按 locale / 内容类型分割
影响低当前 URL 数量较少，无性能问题
建议未来内容超过 5,000 URL 时拆分为 sitemap index + 子 sitemap（按 locale 或 content type 拆分）。
